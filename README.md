# DaVinci Resolve AI Keywording Agent

Python agent for reading and writing keyword metadata on clips in DaVinci Resolve, with an AI-assisted pipeline.

## Current Status

- `v0.12.0` is the current release.
- Browser-based UI (`app.py`) for reading, editing, and receiving keyword suggestions for the selected clip.

## Requirements

- Python 3.10+
- DaVinci Resolve installed and running
- External scripting enabled in Resolve (`Preferences → System → General`)
- `ffmpeg` on your system (used to extract thumbnail and sample frames):

```bash
brew install ffmpeg
```

- [Ollama](https://ollama.com) with the `llava` model (used for AI keyword suggestions):

```bash
brew install ollama
ollama pull llava
```

- Python dependencies:

```bash
pip install -r requirements.txt
```

- `face_recognition` (optional — required for identity recognition; needs dlib):

```bash
pip install face_recognition
```

## Run

Start Ollama (required for AI suggestions; the rest of the UI works without it):

```bash
ollama serve
```

Verify the model is available (you should see `llava` listed):

```bash
ollama list
```

If it's missing, pull it:

```bash
ollama pull llava
```

Then start the app:

```bash
python app.py
```

Then open `http://localhost:5000` in your browser.

## What It Does

- Connects to an open DaVinci Resolve instance through the official scripting API.
- Finds the selected clip from the current timeline video item, falling back to the Media Pool selection.
- Displays a thumbnail of the clip (extracted from the proxy file — original media is never touched), the clip name, and its keywords in the browser.
- **Refresh** fetches live data from Resolve without reloading the page.
- Each keyword tag has a × button; clicking it opens an inline confirmation modal.
- **Remove** deletes the keyword from the list; a **Save** button then appears.
- **Save** writes the updated keyword list back to Resolve, with a brief "Saved" confirmation on success.
- **Prev / Next** buttons navigate between clips in the current Media Pool folder, ordered by Date Created.
- **Suggested** keywords appear as purple buttons below the keyword list. Up to 3 keywords are recommended based on frequency across clips recorded on the same calendar day in the same folder. Clicking a suggestion adds it to the pending list; Save writes it to Resolve.
- **AI Suggested** keywords appear as teal editable rows below the proximity suggestions. Up to 3 keyword phrases are generated by the llava VLM running locally via Ollama, based on 5 sampled frames from the clip. The section is hidden if Ollama is not running.
- **Add keyword** input at the bottom accepts free-text keywords. While typing, a dropdown shows matching keywords already used elsewhere in the project (case-insensitive substring match). Arrow keys navigate the list; Enter or click commits the keyword.
- **Detected Identities** panel on the right side of the page shows one card per detected person in the clip. Each card shows a face crop, a status badge (known / low confidence / unknown), an assign input pre-filled when a match is found, and an "Add as keyword" toggle. Click **Apply identities to keywords** to write confirmed names to the clip. Confirmed identities are saved to a local registry and recognised automatically on future clips. The panel is hidden if `face_recognition` is not installed or no proxy is available.
- **Keyboard shortcuts** (active whenever focus is not in a text field):

| Key | Action |
|-----|--------|
| `←` or `↑` | Previous clip |
| `→` or `↓` | Next clip |
| `s` | Save |

## Proxy-only policy

All ffmpeg operations (thumbnail, AI suggestion frames, identity detection frames) run against the proxy file (`Proxy Media Path`) only. If no proxy exists, the thumbnail shows "No proxy available" and AI suggestions and identity detection are skipped. Original media files are never read.

## API

`GET /api/clip` — returns the selected clip name, keywords, proxy path, and proximity suggestions.

`GET /api/clip/thumbnail` — returns a PNG thumbnail of the selected clip (`204` if unavailable).

`GET /api/clip/suggestions` — returns up to 3 proximity-based keyword suggestions.

`GET /api/clip/ai-suggestion` — returns up to 3 AI-generated keyword phrases from the llava VLM.

`POST /api/clip/keywords` — writes an updated keyword list to the selected clip.

`POST /api/clip/navigate` — navigates to the next or previous clip in the current folder.

`POST /api/clip/detect-identities` — runs face detection on proxy frames and returns detected faces with match status.

`GET /api/clip/face-crop` — returns the JPEG face crop for a given face token.

`GET /api/identities` — returns the list of known identities.

`POST /api/identities/confirm` — commits identity assignments and updates the local registry.
